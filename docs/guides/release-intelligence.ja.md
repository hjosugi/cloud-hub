# クラウドリリースを設計・運用の観点で読む

新機能の件数ではなく、既存アーキテクチャの前提が変わるリリースを拾う。

## 8つの確認

1. ステージ：プレビュー / GA / 廃止 / 引退 / セキュリティ修正
2. 範囲：グローバル / リージョン / ゾーン / アカウントなど
3. 既存リソース：自動適用かオプトインか、新規だけか
4. データプレーン：トラフィック / データ形式 / 一貫性が変わるか
5. コントロールプレーン：API、IAM、ポリシー、デフォルトが変わるか
6. 運用：メトリクス、ログ、クォータ、ランブック、バックアップが変わるか
7. 経済性：価格設定、コミットメント、イーグレス、ライセンスが変わるか
8. クロスクラウド：他クラウドの同等機能と責任範囲はどう違うか

## ステージ別の行動

| ステージ | 設計者 | 運用者 |
|---|---|---|
| プレビュー | アーキテクチャオプションとして記録、プロダクション依存は避ける | サンドボックス検証、SLA / サポート / リージョンを確認 |
| GA | 既存ADRの制約を再評価 | メトリクス、クォータ、コスト、ランブックを検証 |
| デフォルト変更 | セキュリティ / 可用性前提の差分を確認 | 既存 / 新規リソースの適用範囲をインベントリで特定 |
| 廃止 | 置換とマイグレーションアーキテクチャを決定 | 締切、所有者、対象リソース、ロールバックをIssue化 |
| セキュリティ | エクスポージャーと補償コントロールを評価 | パッチ / 緩和策、証拠、完了確認 |

## クラウドごとの読み方

### AWS

サービス単位のリリースが多い。リージョンの利用可能性、IAMアクション、サービスクォータ、既存リソースへの適用、CloudFormation / APIサポートを確認する。

### Azure

プレビュー→GAだけでなく、Azureポリシー、APIバージョン、SKUの廃止、Entra / Defenderとの統合変更を見る。サブスクリプション / マネージドグループへの展開に伴うポリシー影響を確認。

### Google Cloud

プロダクトリリースノートは日単位で複数サービスがまとめられる。ローンチステージ、グローバル / リージョナル範囲、クォータ、データ / MLモデルのバージョン、廃止APIを確認。

### OCI

データベースバージョン、リージョン / アドバンスドドメインの可用性、シェイプ、ネットワーク、IAMポリシー例、コンソール / API / CLIサポートの差を見る。年間認証コードは実務リリースと分離する。
