## Recording Mistaken Questions

Do not manage correct answers or overall scores. Only mistakes, guesses that happened to be correct, and questions narrowed down to two choices are compressed into these four elements:

1. What did I overlook?
2. Why did I choose the wrong answer?
3. What are the key requirement words that fixed the correct answer?
4. One-line rule for similar questions in the future

Add one line to the mistake question notebook for each exam, then read only the judgment criteria from the corresponding notes. The next day, three days later, and seven days later, explain without looking at the one-line rule. If you can explain three times in a row, move it to the "mastered" section.

| Code | Cause | Correction Method |
|---|---|---|
| K | Knowledge | Memorize the role and constraints of the service in one line |
| R | Requirement | Pick out requirement words like `MOST`, `LEAST`, sync/async, RPO, etc. |
| C | Confusion | Create comparison tables of similar services |
| T | Trap | Record traps such as over-design, homemade solutions, fine-tuning myths, etc. |
| M | Multiple response | Judge each option as a separate true/false |
| F | Freshness | Correct outdated names or discontinued specifications with official info |

## 20-Minute Review Units

- 5 minutes: Hide the one-line rule for mistaken questions and explain
- 10 minutes: Review the corresponding comparison table or scenario drill
- 5 minutes: Update only the lines you couldn't explain

Only watch new videos from the beginning if there are no relevant points in the existing guides.
