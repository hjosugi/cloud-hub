## 間違い問題の記録

正解した問題や総合点は管理しない。誤答、勘で正解、2択までしか絞れなかった問題だけを、次の4要素に圧縮する。

1. 何を見落としたか
2. なぜ誤答を選んだか
3. 正解を固定した要件語は何か
4. 次に同型問題が出たときの一行ルール

試験別の間違い問題ノートへ1行追加し、対応するノートの判断軸だけを読む。翌日、3日後、7日後に一行ルールを見ずに説明し、3回連続で説明できたら定着済みに移す。

| コード | 原因 | 修正方法 |
|---|---|---|
| K | Knowledge | サービスの役割・制約を1行で覚える |
| R | Requirement | `MOST`、`LEAST`、同期/非同期、RPO等の要件語を拾う |
| C | Confusion | 似たサービスの比較表を作る |
| T | Trap | 過剰設計、自作、fine-tuning万能論などの罠を記録する |
| M | Multiple response | 各選択肢を独立した○×として判定する |
| F | Freshness | 古い名称・廃止済み仕様を公式情報で補正する |

## 20分の復習単位

- 5分: 間違い問題の一行ルールを隠して説明
- 10分: 対応する比較表またはシナリオ演習
- 5分: 説明できなかった行だけ更新

新しい動画を最初から見るのは、既存のガイドに該当論点がない場合だけにする。
