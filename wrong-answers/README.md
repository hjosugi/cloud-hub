# 間違い問題だけを復習する

正解した問題や総合点は管理しない。迷った問題と間違えた問題だけを、次の4要素に圧縮する。

1. 何を見落としたか
2. なぜ誤答を選んだか
3. 正解を固定した要件語は何か
4. 次に同型問題が出たときの一行ルール

## 使い方

1. 試験別ファイルの表へ1行追加する。
2. `notes/` の対応する問題番号と判断軸だけを読む。
3. 翌日、3日後、7日後に「一行ルールを見ずに説明」する。
4. 3回連続で説明できたら状態を `定着` にする。

記録対象は、誤答、勘で正解、2択までしか絞れなかった問題。確信を持って正解した問題は記録しない。

## 誤答原因コード

| コード | 原因 | 修正方法 |
|---|---|---|
| K | Knowledge | サービスの役割・制約を1行で覚える |
| R | Requirement | `MOST`、`LEAST`、同期/非同期、RPO等の要件語を拾う |
| C | Confusion | 似たサービスの比較表を作る |
| T | Trap | 過剰設計、自作、fine-tuning万能論などの罠を記録する |
| M | Multiple response | 各選択肢を独立した○×として判定する |
| F | Freshness | 古い名称・廃止済み仕様を公式情報で補正する |
